# HumanKey — посадочный сайт

Статический лендинг (HTML + CSS, без зависимостей). Готов к деплою на GitHub Pages, Netlify, Cloudflare Pages, Vercel или любой хостинг под статику.

## Структура

```
site/
├── index.html        — главная (hero, фичи, как работает, скачать, обещание)
├── privacy.html      — Политика конфиденциальности (РФ-формулировки)
├── terms.html        — Пользовательское соглашение
├── styles.css        — Deep Cosmos палитра, без внешних зависимостей
├── favicon-16/32/96.png  — фавиконы
├── icon-180.png      — apple-touch-icon
├── icon-192/512.png  — PWA-готовые иконки
├── og-image.png      — превью 1200×630 для Telegram/VK/соцсетей
├── .nojekyll         — отключает Jekyll-обработку на GitHub Pages
├── deploy.ps1        — автоматический скрипт деплоя (Windows PowerShell)
└── README.md         — этот файл
```

## Что сделать перед первой публикацией

Открой в редакторе и поправь под себя:
- [ ] **`privacy.html`** — реквизиты ИП (ФИО, ОГРНИП), реальный email
- [ ] **`terms.html`** — то же
- [ ] **`index.html`** — `feedback@humankey.app` → ваш реальный почтовый ящик
- [ ] **`index.html`** — `og:url` указать на реальный домен после покупки

## Вариант А: GitHub Pages (бесплатно, 5 минут)

### Шаг 1. Создать публичный репо на github.com

1. Зайди на github.com → New repository
2. Название: например `humankey-site` (любое)
3. **Public** (это важно для бесплатного Pages)
4. **Don't** добавляй README/lic/gitignore — у нас уже есть
5. Create

### Шаг 2. Деплой одной командой (Windows)

В PowerShell, из папки `d:\Project\HumanKey\site\`:

```powershell
.\deploy.ps1
```

Скрипт спросит URL репо (один раз), сделает init/commit/push.

При следующих обновлениях — просто:

```powershell
.\deploy.ps1 "обновил тексты"
```

### Шаг 3. Включить Pages в репо

После первого пуша:

1. На GitHub: твой репозиторий → **Settings** → **Pages**
2. Source: **Deploy from a branch**
3. Branch: **main** / Folder: **/ (root)**
4. Жми **Save**

Через 1-2 минуты сайт будет жить по адресу:
```
https://USERNAME.github.io/REPO-NAME/
```

(где USERNAME и REPO-NAME — ваши)

## Вариант Б: Netlify (drag-and-drop, без терминала)

1. Регистрируйся на netlify.com (можно через GitHub)
2. На дашборде: **Sites → Add new site → Deploy manually**
3. Перетащи папку `site/` в окно
4. Готово — сайт на случайном поддомене `humankey-xxx.netlify.app`
5. Для своего домена: **Site settings → Domain management → Add custom domain**

Плюс Netlify: каждый коммит в репо автоматически деплоит. Минус: чуть сложнее в первоначальной настройке.

## Подключение собственного домена `humankey.app`

После того как сайт живёт на github.io субдомене:

### 1. Купи домен

- **reg.ru** — `~1500 ₽/год` для `.app` (НДС включён, оплата картой РФ)
- **namecheap.com** — `$15/год` (валютная карта)
- **porkbun.com** — `$10/год` (самый дешёвый, валютная карта)

Выбирай тот, где можешь оплатить.

### 2. В DNS настройках домена добавь записи

```
Тип   Хост    Значение
A     @       185.199.108.153
A     @       185.199.109.153
A     @       185.199.110.153
A     @       185.199.111.153
CNAME www     USERNAME.github.io
```

### 3. На GitHub: репо → Settings → Pages → Custom domain

Впиши `humankey.app` (без https://), Save.

### 4. Включи HTTPS

Через 15-30 минут (после выпуска SSL-сертификата от Let's Encrypt) появится галочка **Enforce HTTPS** — поставь её.

Готово. Сайт по `https://humankey.app`.

## Если что-то не так

- **Сайт не открывается** — проверь, что репо публичный (Public), и Pages включён
- **CSS / иконки не грузятся** — проверь относительные пути (нет `/`-абсолютных)
- **Кириллица крякозябрами** — проверь `<meta charset="UTF-8">` в HTML
- **HTTPS не включается** — подожди ещё 15-30 минут, сертификат выпускается отдельно после DNS пропагирования
